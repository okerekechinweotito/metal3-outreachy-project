Metal³ - The Future of Infrastructure
Deployment is Here!

Metal³ is an open-source project collaboration between OpenStack Foundation and Kubernetes Community. Metal³ is itself a Kubernetes application, so it runs on Kubernetes, and uses Kubernetes resources and APIs as its interface.

Metal³.io Overview

Metal³ makes it easier for organizations to deploy and manage their applications at scale. 

        
 Metal³ Prerequisites
          -System with CentOS 8 Stream or Ubuntu 20.04
          -Bare metal preferred, as we will be creating VMs to emulate bare metal hosts
          -Run as a user with passwordless sudo access
          -Minimum resource requirements for the host machine: 4C CPUs, 16 GB RAM memory
        
          Operations with Metal³
          -Discover hardware inventory and configure BIOS / RAID settings on your hosts
          -Optionally clean a host's disks as part of provisioning
          -Install and boot operating system image of choice
          -Deploy Kubernetes with non-disruptive rolling upgrades.
          -Auto-fix failed nodes by rebooting or removing them from the cluster.

Why Metal³.io

The Metal³.io project is a promising open-source initiative that aims to revolutionize the way infrastructure management is done in the Kubernetes ecosystem. By leveraging the capabilities of Kubernetes, Metal³ is building a Kubernetes native API for managing bare metal hosts via a provisioning stack that runs on Kubernetes.

Uses Ironic

Metal³ builds on the work of other open-source tools for bare metal host provisioning, such as Ironic, and extends their capabilities by providing a Kubernetes-native API. This approach enables users to manage their bare metal infrastructure just like they manage their applications, using familiar tools and interfaces, and leveraging the scalability and flexibility of Kubernetes.

Cluster API

One of the key features of Metal³ is its integration with the Kubernetes cluster-API project. This integration allows Metal³ to be used as an infrastructure backend for Machine objects from the Cluster API, making it easier for users to manage and deploy their bare metal hosts.

Scalable and Open-Source

With Metal³, users can deploy and manage their bare metal infrastructure at scale, similar to how cloud providers manage virtual machines. This solution is ideal for use cases that require high-performance computing, edge computing, and traditional data centers. By providing a scalable, open-source solution for bare metal infrastructure management, Metal³ is empowering users to take control of their infrastructure and leverage the full potential of Kubernetes.

Components

Baremetal Operator (BMO)

Baremetal operator is a Metal³ component that automates the deployment and management of bare metal infrastructure. It uses Ironic, an OpenStack project, as the underlying provisioning and management tool for bare metal servers. It is the main component that interfaces with the Ironic API for all operations needed to provision bare-metal hosts... Read more

Cluster API Provider Metal³ (CAPM3)

The Cluster API Provider integrates Metal³ with the Cluster API framework which is used for managing Kubernetes clusters in a declarative manner. It is a comprehensive reference manual outlining the steps needed to optimize computer resources for various tasks... Read more

BareMetalHost

Baremetalhost is a specialized resource that embodies a physical server and all its relevant inventory data. It serves a similar function to a name tag that is placed on each computer to identify its type and capabilities... Read more

Hardware Classification Controller (HCC)

The Hardware Classification Controller (HCC) is a specialized controller that categorizes BareMetalHosts into custom-defined groups according to their hardware specifications. It functions similarly to a sorting machine, enabling users to arrange their computers into distinct groups based on their respective name tags... Read more

IP Address Manager (IPAM)

The IP address manager is a crucial component responsible for assigning unique IP addresses to BareMetalHosts from a pool of available addresses. This system functions similarly to a phone book that provides each computer with a distinctive identification number, enabling communication between them.... Read more

Metal³ Hardware Inventory Record (M3HIR)

The Metal³ Hardware Inventory Record is a specialized resource designed to store hardware inventory data gathered from BareMetalHosts. It functions similarly to a diary, keeping a detailed log of information regarding your computers and their activities... Read more

    Goals

  • To provide a framework that is scalable, vendor-neutral, and accessible to users of any skill level.

  • Metal³ aims to reduce the complexity and cost of managing bare metal infrastructure by leveraging Kubernetes to automate many of the common tasks associated with managing servers.

    Missions

  • To make the deployment, configuration, and management of bare metal infrastructure as easy as cloud infrastructure.

  • Metal3 aims to provide a consistent and declarative API for managing bare metal infrastructure while also being open source and community driven
